Introducing Morphine: The Virtualization Revolution

In the world of PE packing, the battle lines are constantly shifting. what worked yesterday—simple file-dropping and basic XOR loops—is now trivial for modern heuristic engines and EDRs to flag. to stay ahead, we have to stop playing by the processor's rules.

today, we are announcing a fundamental shift in the TinyLoad architecture: The Morphine VM Engine (coming in v3).

Moving Beyond the Instruction Set

traditional packers rely on native x86/x64 instructions to handle payload decryption. while effective for a time, these patterns are easily "fingerprinted." if an analyst sees a specific loop with a specific XOR constant, they’ve already won.

Morphine changes the game by introducing a custom bytecode virtual machine directly into the TinyLoad stub.

How Morphine Works

instead of running native assembly, the TinyLoad stub will embed a lightweight interpreter. the logic for decrypting your payload is "lifted" into a custom language that only this specific interpreter understands.

  • Instruction Set Randomization: every time you pack a file, the "opcodes" are reshuffled.
  • Decoupled Control Flow: because the logic is virtualized, standard disassemblers like IDA or Ghidra cannot automatically trace the decryption process.
  • In-Memory Purity: combined with our v2 manual-mapping technology, the payload remains virtualized until the very moment it is executed in-memory.

What is this

TinyLoad is a simple tool for packing Windows executables. it compresses the input file and can optionally apply rolling XOR encryption to the payload. when the packed executable runs, it extracts the original to a temp directory, runs it, waits for it to finish, then cleans up.

the whole thing is one .cpp file with no external dependencies. makes it easy to integrate or just use as a reference.

Usage

TinyLoad.exe --i app.exe --c
TinyLoad.exe --i app.exe --o packed.exe --xor --c
TinyLoad.exe --i game.exe --xor
Flag What it does
--i <file> input executable to pack
--o <file> output path, defaults to inputname_packed.exe
--xor encrypt payload with rolling XOR, makes static analysis harder
--c compress with LZ77

How compression works

custom hash-chain LZ77 with a 64KB sliding window and deep chain search. uses lazy match evaluation to pick better matches. compression runs on the raw PE first, then XOR is applied on top so patterns in the compressed stream are also hidden from static analysis.
